# Decisions And Options (Q&A)

Last updated: 2026-03-26

Q: What is currently locked?
A:
1. Checkout mode baseline is hosted overlay.
2. Custom checkout is out of current scope.
3. PWA scope is WordPress front-end only; checkout/payment remain in ROLLER.
4. Identity baseline is ROLLER online accounts (email and/or phone).

2. https://mysupport.roller.software/hc/en-us/articles/5517737900687-Create-your-online-checkout
3. https://mysupport.roller.software/hc/en-us/articles/7382787636879-Sign-in-and-use-online-accounts

Q: What major loyalty launch options are available?
A:
1. Option 1: ROLLER membership and marketing automation (no stamp UI).
2. Option 2: Patch points loyalty with provider UI (no custom UI).
3. Option 3: Patch loyalty with custom stamp UI in WordPress.
4. Option 4: Visit-based marketing only, no points/stamp UI at launch.
Sources:
1. https://mysupport.roller.software/hc/en-us/sections/5460757524239-Memberships
2. https://mysupport.roller.software/hc/en-us/articles/12557902973839-ROLLER-Patch-Retention
3. https://help.patchretention.com/loyalty-program-rewards-widget

3. Opening-hours integration path for non-ROLLER countries.
4. Product-card data model per country.
5. Final marketing launch scope.
6. Loyalty edge-case rules (refunds, duplicates, same-day repeats, manual corrections).
