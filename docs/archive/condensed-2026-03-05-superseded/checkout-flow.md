# Checkout Flow

Last updated: 2026-03-04

## Default flow (chosen)

1. Visitor lands on WordPress page.
2. Visitor clicks buy button.
3. ROLLER overlay checkout opens.
4. Visitor selects products and pays in ROLLER.
5. Booking confirmation is handled by ROLLER.
6. Tracking events are sent to analytics setup.

## Optional flow (only if needed)

1. Visitor selects online.
2. Visitor pays at venue later.

Note:
1. This pay-later option is documented for legacy checkout; progressive overlay compatibility must be confirmed for your account.
