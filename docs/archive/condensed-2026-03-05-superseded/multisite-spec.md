# WordPress Multisite Spec

Last updated: 2026-03-04

## Topology

1. One multisite network.
2. One site per park.
3. Optional country hub sites for shared campaigns.

## Role model

1. HQ super admin: platform settings, plugins, security, integrations.
2. Country manager: regional campaigns and localization oversight.
3. Park admin: local pages, schedules, local offers.

## Integration pattern

1. Standardized CTA/button blocks open park-specific ROLLER overlay.
2. Common account link pattern to ROLLER account pages.
3. Shared analytics tagging standard across all park sites.

## Build order

1. Phase 1: theme/components + checkout launch blocks.
2. Phase 2: permission matrix + localization patterns.
3. Phase 3: analytics QA and rollout templates for franchises.
