# Mascot Relevance for Loyalty Planning

Date logged: Friday, 2026-03-06
Status: Input for planning, not a locked decision

## Summary

Mascots are relevant for loyalty as an engagement layer.
They are not a replacement for core loyalty mechanics (identity, visit counting, rewards, redemption).

## Where mascots can help

1. Make progress visible and memorable for children and families.
2. Support repeat behavior through story-based milestones.
3. Support first-visit conversion with playful onboarding.
4. Improve emotional value without lowering base prices.

## Practical loyalty use cases

1. Choose your mascot at signup.
2. Mascot-themed milestone journey (for example: 1st, 3rd, 6th, 7th visit moments).
3. Mascot challenges and birthday surprises.
4. Mascot-based communication templates in email/SMS/WhatsApp.

## What this means for ROLLER/Patch requirements

1. Loyalty system must support milestone trigger messages.
2. Loyalty system should support campaign personalization fields (for mascot choice/progress stage).
3. Local parks should be able to run park-specific mascot promotions.
4. Reporting should measure if mascot journeys improve repeat visits.

## Risks and controls

1. Risk: mascot layer adds complexity before core loyalty is stable.
2. Control: phase mascot features after base stamp/milestone flow works reliably.
