# Writing Style Guide (Project Docs)

Use these rules for all future updates.

1. Use plain language first. Avoid jargon when a simple word works.
2. Keep sentences short.
3. Keep only decision-useful information.
4. Separate facts from proposals:
   - `Verified`
   - `Uncertain`
   - `Recommendation`
5. Put dates in full format (for example: Wednesday, 2026-03-11).
6. Keep sections short and scannable.
7. Link sources only for factual claims.
8. If something is unknown, say so clearly.
9. Add context for action points so the object is clear (for example, say "in URLs" instead of leaving it implied).
10. Explain acronyms on first use in parentheses (for example, UTM = Urchin Tracking Module marketing tags).
11. Place content in the 3-level model: `docs/01-concept` (brief), `docs/02-flow` (how it works), `docs/03-implementation` (technical spec).
